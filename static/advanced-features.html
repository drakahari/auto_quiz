Advanced & Experimental Features

Advanced features are optional and intended for experienced users.
Default settings are recommended for most workflows.

Portal Settings

Portal Settings allow you to customize the appearance and behavior of DLMS without affecting core quiz functionality.

Common options include background images, experimental parsing behavior, and optional tooling designed for power users.

Resetting Missed Questions/ Clearing Exam & History Results

DLMS tracks missed questions across quiz attempts to support targeted review and long-term learning. DLMS also maintains a persistent history of completed quiz attempts, including scores, dates, and review data.

If desired, you may reset the missed-question and quiz attempt history. This clears all stored missed-question records while leaving quizzes intact.

Use this option if you want to start fresh after remediation or focused study.

You may choose to clear this history if you want to:

  • Remove old exam results
  • Reset progress tracking
  • Begin a new study cycle

This action affects attempt history only and does not delete quizzes.

Full Database Reset (Factory Reset)

This action is destructive and cannot be undone.

DLMS provides an optional full database reset for users who want to completely wipe all application data.

A full reset permanently deletes:

  • All quizzes
  • All attempt history
  • All missed-question records
  • All stored metadata

This option is intended for:

  • Fresh starts
  • Testing and development
  • System handoff or reuse
DLMS requires explicit confirmation before performing a full reset to prevent accidental data loss.
